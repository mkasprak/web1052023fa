HTML Style Guide

A consistent, clean, and tidy HTML code makes it easier for others to read and understand your code.

Consistent HTML Code Guidelines:

  • Declare document type as the first line: <!DOCTYPE html>
  • Use lowercase element names for readability and consistency
  • Close all HTML elements, including those like <p>
  • Use lowercase attribute names for cleanliness
  • Always quote attribute values for readability and spaces
  • Specify alt, width, and height attributes for images
  • Avoid spaces around equal signs in attributes
  • Avoid long code lines for easier readability
  • Use blank lines and indentation thoughtfully

Title Element:

Always include <title> element for SEO and browser display

Omitting Elements:

Omitting <html> and <body> tags can cause issues

Omitting <head> is possible, but it's recommended to use it

Close Empty Elements:

Optional to close empty elements, <meta> can be self-closed

lang Attribute:

Include lang attribute in <html> tag for language declaration

Meta Data:

Define <meta charset> and language early in the document

Set viewport using <meta name="viewport"> for responsive design

HTML Comments:

Use short and long comments with proper indentation

Style Sheets:

Link to style sheets with <link rel="stylesheet">

Use consistent formatting for CSS rules

JavaScript in HTML:

Load external scripts with <script src>

Ensure tidy HTML code to avoid JavaScript errors

File Naming and Extensions:

Use lowercase file names for consistency and case sensitivity

Use appropriate file extensions: .html for HTML, .css for CSS, .js for JavaScript

Differences in File Extensions:

No difference between .htm and .html, both treated as HTML by browsers

Default Filenames:

Servers add default filenames like "index.html"

Configure server with multiple default filenames if needed
